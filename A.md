# 一、需求文档（PRD）

## 1. 项目背景与目标

本项目是一个面向大众的「经济学基础知识科普平台」，目标是用通俗、结构化、可视化的方式，帮助非经济学专业用户理解核心经济学概念，并建立基本的经济思维方式。

平台强调三个关键词：**低门槛、可交互、可解释**。用户不需要数学或经济学背景，也能逐步理解供需、价格机制、通胀、货币、市场结构等基础概念。

## 2. 目标用户

* 对经济学感兴趣的普通用户
* 大学生、跨专业学习者
* 想理解宏观经济、金融新闻的非专业人士

## 3. 核心使用场景

* 用户通过网页系统化学习经济学基础知识
* 通过图表、案例、互动问答加深理解
* 借助 AI 助手对概念进行“白话解释”和延展

## 4. 功能需求

### 4.1 内容学习模块

* 经济学基础知识章节化展示
* 每个章节包含：

  * 概念解释（通俗版 / 标准版）
  * 现实案例
  * 简单图表或示意

示例章节：

* 供给与需求
* 价格形成机制
* 通货膨胀与通货紧缩
* 货币与央行
* 市场失灵

### 4.2 可视化模块

* 使用图表展示经济学关系（如供需曲线）
* 支持用户简单调整参数，观察变化结果


## 5. 非功能需求

* 页面响应时间 < 1 秒（核心页面）
* 支持移动端与桌面端
* 内容表达避免专业黑话

# 二、技术文档（Technical Design）

## 1. 总体架构

采用前后端分离架构：

* 前端：Vue 生态
* 后端：Spring Boot 或 Python（FastAPI / Flask）
* 数据库：关系型数据库

## 2. 前端技术方案

### 2.1 技术栈

* Vue 3
* Vite
* TypeScript
* Pinia（状态管理）
* Vue Router
* ECharts（经济学图表可视化）

### 2.2 前端模块划分

* 页面层：

  * 首页（项目介绍）
  * 学习页（章节内容）
  * 图表交互页
  * 用户中心
* 服务层：

  * API 请求封装
* 状态层：

  * 用户状态
  * 学习进度状态

## 3. 后端技术方案

### 3.1 技术选型（两种方案）

**方案：Python**

* FastAPI（推荐）
* SQLAlchemy
* Pydantic
* JWT

## 4. 数据库设计（示意）

### 4.1 核心表

* user（用户表）

  * id

  * password_hash

* chapter（章节表）

  * id

# 三、AI 开发规范文档（AI Usage Guideline）

## 1. AI 使用原则

* AI 只用于**知识解释与辅助理解**
* 不提供任何投资、理财、交易建议
* 不输出带有确定性预测的经济结论

## 2. Prompt 设计规范

* 明确 AI 角色：经济学科普讲解员
* 明确语言要求：通俗、简洁、生活化
* 禁止使用过多数学公式

示例 Prompt 结构：

> 你是一名经济学科普讲解员，请用不超过 150 字，结合生活例子，解释【供给与需求】这一概念。

## 3. 输出内容约束

* 使用比喻、类比
* 避免专业术语堆砌
* 涉及宏观经济时需注明“简化模型”

## 4. 风险控制

* 后端对 AI 返回内容进行关键词检测
* 涉及金融、投资词汇直接拦截或弱化
* 所有 AI 回复需标注：

  > 本内容为经济学科普，不构成任何现实决策建议

## 5. 迭代与评估

* 记录用户常问问题
* 用于优化 Prompt 模板
* 定期人工抽查 AI 输出质量
